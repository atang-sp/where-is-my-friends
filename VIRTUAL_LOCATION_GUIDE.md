# 可选 GPS / 地图模式指南

城市模式已经能完成同城发现。只有希望看到宽泛距离范围的成员，才需要打开“添加隐私距离范围”。

## 三种模式

| 模式 | 浏览器权限 | 保存内容 | 对其他成员显示 |
| --- | --- | --- | --- |
| `city` | 无 | 城市、地区、有效期 | 城市 |
| `gps` | 定位权限 | 添加随机偏移后的坐标 | 城市和粗粒度距离范围 |
| `map` | 无 | 用户主动选择的地图坐标 | 城市和粗粒度距离范围 |

城市和可选地区始终存在。启用 GPS/地图不会改变“只匹配同城成员”的第一层边界。

## 用户流程

1. 先保存城市并进入同城结果页。
2. 选择“添加隐私距离范围”。
3. 使用设备 GPS，或在地图上选择一个公共区域。
4. 保存成功后，列表继续自动刷新，不需要再次点击“查找”。
5. 如果 GPS 被拒绝、超时或不可用，城市模式保持有效。
6. 用户可以随时更新城市、切换模式或删除位置。

地图选择器不做地址解析，也不会把坐标发送给 Nominatim。地图点取消或保存后会从组件状态中清除。

## 地图服务

### OpenStreetMap（默认）

- 不需要 API key。
- Leaflet 1.9.4 JavaScript/CSS 随插件本地分发，只在打开地图模态框时加载。
- 地图瓦片来自 `tile.openstreetmap.org`，并在界面显示 OSM attribution。

生产论坛流量较大时，应遵守 OSM 瓦片使用政策，或将瓦片 URL 改为有容量保障的服务。

### 高德和百度

在管理后台选择供应商并填写对应的浏览器 key：

```yaml
where_is_my_friends_map_provider: "amap" # 或 "baidu"
where_is_my_friends_amap_api_key: "PUBLIC_BROWSER_KEY"
where_is_my_friends_baidu_api_key: "PUBLIC_BROWSER_KEY"
```

- 这些是会发送到浏览器的公开 key，不应填写服务端私钥。
- 必须在供应商控制台设置 HTTP Referer/域名白名单，只允许论坛正式域名和必要的开发域名。
- 插件只返回当前选择供应商的 key；另一个供应商的 key 不会进入页面响应。
- 选择商业供应商但 key 为空时，组件会自动回退到 OpenStreetMap。

## 服务端隐私行为

- GPS：收到坐标后分别加入 `-0.005..0.005` 度随机偏移，再保存。
- 地图：保存用户选中的点，不宣称它是真实位置。
- 坐标只存在数据库和保存请求体中；API 响应、序列化器、日志、分析事件均不包含坐标。
- 距离只返回 `under_5`、`5_to_20` 或 `over_20`，从不返回精确公里数。
- 位置默认 30 天后删除；分析事件 90 天后删除。

## 故障排查

- GPS 失败：检查 HTTPS、浏览器权限和系统定位服务；城市模式不会中断。
- 地图空白：确认论坛 Content Security Policy 允许所选地图脚本和瓦片域名，检查 key 域名限制。
- 商业地图显示 OSM：当前供应商缺少对应 key，属于预期安全回退。
- 输入坐标被拒绝：纬度必须在 `-90..90`，经度必须在 `-180..180`。

验证命令：

```bash
CI=1 d/rake 'plugin:qunit[where-is-my-friends]'
d/rake 'plugin:spec[where-is-my-friends]'
```
